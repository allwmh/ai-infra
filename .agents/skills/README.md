# Agent Skills — 自动记忆系统

让 AI Coding Agent 拥有跨对话的持久记忆，告别"新对话智力回退"。

## 设计理念

传统 AI 对话是无状态的——每次新对话都从零开始。这套 Skill 通过在 workspace 中维护一个 `MEMORY.md` 文件，实现了：

- **自动读档**：新对话自动加载上下文，无需重复介绍项目背景
- **自动存档**：对话过程中自主判断哪些信息值得记住，静默写入
- **防遗忘**：长对话接近上下文窗口限制时，主动提炼结论写入

## Skills

### 🔄 project-resume（上下文恢复）

新对话开始时自动触发，静默执行：

1. 读取 `MEMORY.md` 获取项目状态
2. 扫描目录结构，识别对话外的变化
3. 检查用户当前打开的文件
4. **不做汇报**，直接带着上下文干活

### 💾 session-checkpoint（自动记忆）

模仿 Claude Code 的 Auto Memory 机制，在对话过程中自主判断何时写入 `MEMORY.md`。

**触发条件**（不需要用户指令）：

| 写入                                                | 跳过                |
| --------------------------------------------------- | ------------------- |
| 项目结构变化                                        | 纯代码/文字修改细节 |
| 用户的重要决策                                      | 临时讨论和探索      |
| 项目约定和模式                                      | 已被覆盖的旧信息    |
| 里程碑完成（feature/bugfix/文章定稿且获得正面评价） |                     |
| 用户认可的想法（"这个 idea 很好"等信号）            |                     |
| 长对话的关键结论（防上下文溢出丢失）                |                     |

## 使用方式

1. 将 `.agents/skills/` 目录放入你的项目根目录
2. 确保根目录有 `MEMORY.md`（首次运行时 AI 会自动创建）
3. 正常对话即可，无需特殊指令

兼容任何支持 Agent Skills 的 IDE（Antigravity、Cursor、VS Code + Copilot 等）。

## MEMORY.md 格式

```markdown
# 项目记忆

## 项目概述
（一句话描述）

## 当前结构
（目录和文章的概览表格）

## 决策记录
- YYYY-MM-DD：决策描述

## 项目约定
- 约定 1

## 待办
- [ ] 具体任务
```

保持简洁：总长度 ≤ 100 行，决策记录保留最近 5~10 条，已完成的待办直接删除。
