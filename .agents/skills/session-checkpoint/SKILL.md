# 自动记忆技能

## 核心理念

模仿 Claude Code 的 Auto Memory 机制：**不是定期存档，而是在对话过程中自主判断什么值得记住，随时写入 `MEMORY.md`。**

## 什么时候写入记忆

在对话过程中，当出现以下情况时，**自主判断**是否需要更新 `MEMORY.md`，不需要用户指令：

### 值得记住的（写入）
- 项目结构发生了变化（新增/删除/重命名文件或目录）
- 用户做了重要决策（如内容方向、架构调整、编号规则）
- 发现了项目约定/模式（如文章的写作风格、命名规范）
- 待办事项的产生或完成
- 用户的偏好和习惯
- **里程碑完成**：一个 feature 完成、一个 bug 修复、一篇新文章定稿——尤其是得到用户正面评价（"不错"、"可以"、"就这样"等）时，记录成果和关键决策
- **用户认可的想法/方案**：当用户说"这个 idea 很好"、"就按这个来"等认可性语气时，把被认可的方案要点记下来
- **长对话的关键结论**：当一个问题被反复讨论、对话可能接近或超出上下文窗口限制时，主动把讨论的核心结论和最终共识提炼写入，防止上下文滚动后丢失

### 不值得记住的（跳过）
- 纯粹的代码/文字修改细节（文件本身就是记录）
- 临时的讨论和探索
- 已经被更新的决策覆盖的旧信息

## 如何写入

### 直接更新，不要征求许可

当你判断有值得记忆的信息时，直接更新 `MEMORY.md` 的对应部分。这是静默操作，像人类的"记住了"一样自然，不需要打断工作流去问"要不要更新记忆"。

### MEMORY.md 格式

```markdown
# infra-doc 项目记忆

## 项目概述
（一句话描述）

## 当前结构
（目录和文章的概览表格）

## 决策记录
- YYYY-MM-DD：决策描述

## 项目约定
- 约定 1
- 约定 2

## 待办
- [ ] 具体任务
```

### 保持简洁
- 总长度控制在 **100 行以内**
- 旧的决策记录只保留最近 5~10 条
- 已完成的待办直接删掉
- 被新决策覆盖的旧决策直接删掉

## 用户也可以主动触发

当用户说"存档"、"记一下"、"更新记忆"时，做一次全面的记忆更新。
